# RULES.md（协作执行规范）

## 1. 代码行数约束

* **单个文件**：新建、修改或重构文件时，建议保持 **≤ 300 行**
* **例外情况**（可超出限制）：
  - 自动生成的文件（如 `package-lock.json`、类型定义）
  - 数据文件（如 JSON、SQL schema）
  - 第三方库或框架的配置文件
  - 复杂业务逻辑或完整功能模块（需保持代码完整性）
* **超出处理**：如单个文件过大，优先考虑拆分为多个模块文件

## 2. Git 提交提示（强制）

* 每次**完成代码修改或功能实现**后，必须在回复末尾追加一段**可复制的中文提交信息**
* 该段内容**只包含提交信息本身**，格式如下：

```bash
这里写提交信息（中文、准确、动词开头优先）
```

## 3. 文档创建（默认禁止）

* **未被明确要求时：禁止创建**任何总结 / 报告 / 临时文档

### 3.1 允许创建

* 用户明确要求的文档
* 项目必需的配置或基础文档（如 `README.md`、`package.json` 等）
* 必要的代码文件与测试文件
* 经用户确认的正式项目文档

## 4. 输出风格

* **直接交付结果**：只给必要步骤与关键变更点
* **避免冗余总结**：除非用户明确要求
* 若规则存在冲突或需要新增文档：**优先按更严格约束执行**（先不建文档、先拆分改动）
