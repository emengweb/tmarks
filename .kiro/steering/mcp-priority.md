# MCP 优先使用规则

## Augment Context Engine 优先级

在处理所有代码相关的查询和任务时，**必须优先使用 Augment Context Engine MCP** (`mcp_Augment_Context_Engine_codebase_retrieval`)。

### 强制使用场景

以下场景**必须**首先使用 Augment Context Engine：

1. **代码搜索和定位**
   - 查找函数、类、组件的位置
   - 搜索特定功能的实现
   - 定位相关代码片段

2. **代码理解**
   - 理解代码结构和架构
   - 分析依赖关系
   - 了解功能实现方式

3. **代码分析**
   - 查找使用示例
   - 分析调用关系
   - 理解数据流

4. **任务规划**
   - 在开始任何编码任务前，使用 MCP 了解相关代码
   - 在修改文件前，使用 MCP 查询相关上下文
   - 在回答代码问题前，使用 MCP 获取准确信息


### 执行原则

- **先 MCP，后其他**：任何代码相关任务都先用 MCP 获取上下文
- **语义优先**：优先使用语义搜索而非文本匹配
- **一次查询**：尽量在一次 MCP 调用中获取所有需要的信息
- **详细查询**：向 MCP 提供详细的查询描述以获得最佳结果
