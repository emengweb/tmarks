# AI 批量导入完整实现文档

## 1. 概述

AI 批量导入功能允许用户上传书签文件（HTML/JSON/CSV），通过 AI 自动生成标题、描述和分类（标签或文件夹），并支持编辑后导入到 TMarks 或 NewTab 系统。

### 1.1 核心特性

- **双模式支持**：TMarks（标签模式）和 NewTab（文件夹模式）
- **AI 智能整理**：自动生成标题、描述和分类
- **用户自定义设置**：8 个可配置选项
- **实时进度控制**：暂停、恢复、停止功能
- **编辑确认机制**：导入前可编辑所有数据
- **错误处理**：失败重试、详细错误信息

---

## 2. 整体架构

### 2.1 流程概览

```
上传文件 → 解析规范化 → AI 整理（可选）→ 编辑确认 → 最终导入
```

## 3. 详细流程

### 3.1 步骤 1：文件上传和解析

#### 3.1.1 支持的格式
- **HTML**：浏览器导出的标准书签格式
- **JSON**：TMarks/NewTab 导出格式
- **CSV**：自定义 CSV 格式

#### 3.1.2 解析流程
1. 用户选择文件格式和上传文件
2. `parseBookmarksFile()` 解析文件内容
3. `normalizeBookmarksWithStream()` 规范化和验证 URL
4. 实时显示解析进度和结果统计

#### 3.1.3 数据结构
解析后生成 `ParsedBookmark[]`：
- `url`: 书签 URL
- `title`: 原始标题
- `description`: 原始描述
- `tags`: 原始标签数组
- `folder`: 原始文件夹（如有）
- `created_at`: 创建时间

---

### 3.2 步骤 2：AI 智能整理

#### 3.2.1 用户配置选项

**基础设置：**
- **并发数量**：1-10，控制同时处理的 URL 数量（默认 3）
- **批处理延迟**：200ms/500ms/1000ms，避免 API 限流

**内容设置：**
1. **标题长度**
   - 简短：5-15 字
   - 适中：10-30 字（默认）
   - 详细：20-50 字

2. **描述详细度**
   - 极简：10-20 字
   - 简短：20-50 字（默认）
   - 详细：50-100 字

3. **标签数量**（仅 TMarks 模式）
   - 最少：1-3 个（默认 2）
   - 最多：3-8 个（默认 5）

4. **语言偏好**
   - 中文：翻译外文为中文
   - 英文：翻译中文为英文
   - 混合：保持原语言（自动检测浏览器语言）

5. **AI 创造性**
   - 保守（0.3）：结果更一致
   - 平衡（0.7）：推荐
   - 创新（1.0）：结果更多样

6. **风格自定义**
   - AI 自动：完全交给 AI 决定
   - 自定义：用户描述分类习惯

#### 3.2.2 AI 处理流程

**批量处理机制：**
1. 按并发数分批处理 URL
2. 每批并发调用 AI API
3. 批次间延迟指定时间
4. 实时更新进度和统计

**单个 URL 处理：**
1. 构建 AI 提示词（包含用户设置）
2. 调用 AI API（支持多种 Provider）
3. 解析 AI 返回的 JSON
4. 标记新标签（TMarks）或新文件夹（NewTab）
5. 失败时创建基础书签

**提示词构建逻辑：**

**TMarks 模式（生成标签）：**
- 包含已有标签库（最多 100 个）
- 根据语言偏好调整翻译规则
- 根据语言调整标签长度规则（中文 2-4 字，英文 1-3 词）
- 包含用户自定义风格（如有）
- 指定标签数量范围
- 要求优先匹配已有标签

**NewTab 模式（生成文件夹）：**
- 包含已有文件夹列表（最多 7 个）
- 根据语言偏好调整翻译规则
- 根据语言调整文件夹名长度规则
- 包含用户自定义分类偏好（如有）
- 达到 7 个上限时强制从已有文件夹选择
- 提供常见文件夹参考

**AI 返回格式：**

TMarks 模式：
```
{
  "title": "生成的标题",
  "description": "生成的描述",
  "tags": ["标签1", "标签2", "标签3"]
}
```

NewTab 模式：
```
{
  "title": "生成的标题",
  "description": "生成的描述",
  "folder": "文件夹名"
}
```

#### 3.2.3 进度控制

**实时进度显示：**
- 当前进度：已处理 / 总数
- 进度条：百分比可视化
- 统计信息：已处理、成功、失败数量

**控制功能：**
- **暂停**：停止发送新请求，等待当前批次完成
- **恢复**：继续处理剩余 URL
- **停止**：中止处理，为剩余 URL 创建基础书签

#### 3.2.4 错误处理

**失败处理：**
- 记录失败 URL 和错误信息
- 为失败 URL 创建基础书签（使用原 URL 作为标题）
- 显示前 10 个错误详情
- 提供"重试失败 URL"功能

**重试机制：**
- 仅重试失败的 URL
- 使用相同的用户设置
- 更新进度和统计信息

---

### 3.3 步骤 3：编辑确认

#### 3.3.1 数据结构

AI 整理后生成 `EditableBookmark[]`：
- `url`: 书签 URL
- `title`: AI 生成的标题
- `description`: AI 生成的描述
- `tags`: 标签数组（TMarks 模式）
  - `name`: 标签名
  - `isNew`: 是否为新标签
  - `confidence`: 相关性置信度
- `folder`: 文件夹名（NewTab 模式）
- `isSelected`: 是否选中导入
- `isSkipped`: 是否跳过

#### 3.3.2 编辑功能

**表格功能：**
- 分页显示（每页 50 条）
- 搜索过滤（URL、标题、标签）
- 全选/反选
- 统计信息（总数、筛选数、新标签数、已选数）

**单个书签编辑：**
- 编辑标题（点击编辑图标）
- 添加/删除标签（TMarks 模式）
- 查看文件夹（NewTab 模式，只读）
- 选择/取消选择

**批量操作：**
- 批量选择/取消
- 批量添加标签（TMarks 模式）

**显示差异：**
- **TMarks 模式**：显示标签列，新标签用蓝色高亮
- **NewTab 模式**：显示文件夹列，用紫色标签显示

---

### 3.4 步骤 4：最终导入

#### 3.4.1 TMarks 导入

**数据转换：**
1. 过滤选中且未跳过的书签
2. 转换为 `ParsedBookmark[]` 格式
3. 提取标签名称（去除 isNew 等元数据）

**API 调用：**
1. 构建 TMarks JSON 格式数据
2. 调用 TMarks API `/api/v1/import`
3. 传递导入选项（缩略图、快照等）

**返回结果：**
- 成功数量
- 失败数量
- 跳过数量
- 错误详情
- 创建的书签和标签列表

#### 3.4.2 NewTab 导入

**数据处理：**
1. 过滤选中且未跳过的书签
2. 按文件夹分组

**Chrome Bookmarks API 调用：**
1. 确保 NewTab 根文件夹存在
2. 获取已有文件夹列表
3. 为每个文件夹：
   - 查找或创建文件夹
   - 批量创建书签到该文件夹
4. 记录创建的文件夹和书签数量

**返回结果：**
- 导入数量
- 总数量
- 失败数量
- 错误详情（前 10 个）
- 创建的文件夹列表
- 文件夹数量

## 4. 技术实现

### 4.1 核心服务：BookmarkOrganizer

**职责：**
- 批量处理 URL
- 管理并发和延迟
- 构建 AI 提示词
- 解析 AI 返回
- 进度回调
- 状态控制（暂停/恢复/停止）

**关键方法：**

**`organizeUrls(urls, options, onProgress)`**
- 输入：URL 数组、配置选项、进度回调
- 输出：`OrganizeResult`（书签数组、成功数、失败数、错误列表）
- 流程：分批处理 → 并发调用 → 收集结果 → 更新进度

**`processUrl(url, options)`**
- 输入：单个 URL、配置选项
- 输出：`EditableBookmark`
- 流程：构建提示词 → 调用 AI → 解析返回 → 标记新标签

**`buildTMarksPrompt(url, options)`**
- 构建 TMarks 模式的 AI 提示词
- 包含已有标签、用户设置、语言规则

**`buildNewTabPrompt(url, options)`**
- 构建 NewTab 模式的 AI 提示词
- 包含已有文件夹、用户设置、语言规则、7 个上限提示

**`parseAIResponse(content)`**
- 提取 JSON 部分
- 解析标题、描述、标签/文件夹
- 容错处理

**状态控制方法：**
- `pause()`: 设置暂停标志
- `resume()`: 清除暂停标志
- `abort()`: 设置中止标志
- `getStatus()`: 获取当前状态

### 4.2 AI 客户端：callAI

**支持的 Provider：**
- OpenAI
- Anthropic
- Google Gemini
- DeepSeek
- 自定义 API

**调用参数：**
- `provider`: AI 提供商
- `apiKey`: API 密钥
- `model`: 模型名称（可选）
- `apiUrl`: 自定义 API 地址（可选）
- `prompt`: 提示词
- `temperature`: 创造性参数
- `maxTokens`: 最大 token 数

**返回格式：**
- `content`: AI 返回的文本内容
- `usage`: token 使用统计（如有）

### 4.3 数据流转

**类型定义：**

**`OrganizeOptions`**（AI 整理配置）
- 基础：provider, apiKey, model, apiUrl
- 并发：concurrency, batchDelay
- 模式：mode (tmarks/newtab)
- 已有数据：existingTags, existingFolders
- 用户设置：titleLength, descriptionDetail, tagCountMin, tagCountMax, language, temperature
- 风格：tagStyle

**`OrganizeProgress`**（进度信息）
- current: 当前进度
- total: 总数
- status: 状态描述
- successCount: 成功数
- failedCount: 失败数

**`OrganizeResult`**（整理结果）
- bookmarks: EditableBookmark[]
- successCount: 成功数
- failedCount: 失败数
- errors: 错误列表

**`EditableBookmark`**（可编辑书签）
- url, title, description
- tags: { name, isNew, confidence }[]
- folder: string
- isSelected, isSkipped
